# 功能设计

## 模块概览

MVP 原则：能用、够用、不复杂。所有功能均为 ffmpeg 原生直接支持。

| 功能模块 | 具体功能 | 实现方式 |
|----------|----------|----------|
| 文件导入 | 拖拽 / 文件选择，自动读取分辨率、时长、编码、帧率、码率 | ffprobe |
| 裁剪 | 按时间码截取片段；快速模式（-c copy）/ 精确模式（重编码） | ffmpeg 原生 |
| 格式转换 | 输出 mp4 / mkv / mov / webm | ffmpeg 原生 |
| 分辨率缩放 | 预设 4K / 1080p / 720p / 480p，或自定义 | ffmpeg -vf scale |
| 音频处理 | 提取音频（mp3/aac/wav）、去除音频 | ffmpeg 原生 |
| GPU 检测 | 启动时自动探测，转码优先使用硬件编码 | Rust 核心库 |
| 进度显示 | 实时进度百分比、速度、预计剩余时间 | 解析 ffmpeg stderr |
| 批量队列 | 多文件排队处理，串行执行 | Rust 任务队列 |
| 输出控制 | 自定义输出路径、预估文件大小、覆盖提示 | 应用层逻辑 |
| 错误处理 | 友好错误提示 + 折叠查看原始 ffmpeg 日志 | 应用层逻辑 |

---

## 模块一：文件导入

### 功能列表

- [x] 支持拖拽导入文件
- [x] 支持文件选择对话框导入
- [x] 自动读取分辨率、时长、编码、帧率、码率（通过 ffprobe）

### 业务逻辑

```
用户拖拽/选择文件 → 调用 ffprobe 解析 → 展示文件元信息（分辨率、时长、编码、帧率、码率）
```

### 边界情况

- 文件格式不被 ffmpeg 支持 → 显示友好错误提示
- 文件损坏或无法读取 → 提示文件异常，可展开查看 ffprobe 原始输出
- 超大文件 → ffprobe 仍可正常读取元信息，不影响导入

---

## 模块二：视频裁剪

### 功能列表

- [x] 按时间码（开始/结束）截取片段
- [x] 快速模式（-c copy，不重编码，速度快但首帧可能不精确）
- [x] 精确模式（重编码，帧精确但耗时更长）

### 业务逻辑

```
用户设置起止时间 → 选择模式 → 快速模式: ffmpeg -ss START -to END -c copy
                           → 精确模式: ffmpeg -ss START -to END -c:v ENCODER -c:a ENCODER
```

### 边界情况

- 起止时间超出视频时长 → 自动裁剪到视频边界
- 起止时间相同或反转 → 显示错误提示
- 快速模式下裁剪不精确 → 提示用户切换精确模式

---

## 模块三：格式转换

### 功能列表

- [x] 输出格式：mp4、mkv、mov、webm
- [x] 自动选择兼容编码器

### 业务逻辑

```
用户选择输出格式 → 根据格式和 GPU 能力选择编码器 → ffmpeg 转码输出
```

### 边界情况

- 容器格式与编码不兼容 → 自动选择兼容编码组合
- webm 格式无 GPU 加速 → 回退 CPU 编码

---

## 模块四：分辨率缩放

### 功能列表

- [x] 预设分辨率：4K / 1080p / 720p / 480p
- [x] 自定义分辨率
- [x] 自动保持宽高比（默认）

### 业务逻辑

```
用户选择目标分辨率 → ffmpeg -vf scale=W:H → 输出
```

### 边界情况

- 目标分辨率高于源分辨率 → 允许但提示可能无意义
- 自定义非标准分辨率 → 传入 ffmpeg，由 ffmpeg 处理奇数尺寸问题

---

## 模块五：音频处理

### 功能列表

- [x] 提取音频（mp3 / aac / wav）
- [x] 去除音频（仅保留视频流）

### 业务逻辑

```
提取音频: ffmpeg -i INPUT -vn -acodec CODEC OUTPUT
去除音频: ffmpeg -i INPUT -an OUTPUT
```

### 边界情况

- 源文件无音频流 → 提取音频时提示无音频轨
- 去除音频后输出 → 正常输出无声视频

---

## 模块六：GPU 检测

### 功能列表

- [x] 启动时自动探测硬件编码能力
- [x] Windows：h264_nvenc（NVIDIA 独显）→ h264_qsv（Intel 核显）→ 回退 CPU
- [x] macOS：hevc_videotoolbox / h264_videotoolbox → 回退 CPU
- [x] 探测结果缓存，UI 显示当前加速状态
- [x] 用户可手动切换编码器

### 业务逻辑

```
应用启动 → 按优先级依次测试编码器 → 缓存结果 → UI 显示加速状态
解码同步启用: -hwaccel cuda (NVIDIA) / -hwaccel videotoolbox (Apple)
```

### 边界情况

- 无独显 → 探测失败，回退 CPU，UI 显示"软件编码"
- GPU 驱动异常 → 探测超时，回退 CPU

---

## 模块七：进度显示

### 功能列表

- [x] 实时进度百分比
- [x] 当前处理速度
- [x] 预计剩余时间

### 业务逻辑

```
ffmpeg 输出 stderr → 解析 progress=speed= 和 out_time_us= → 计算百分比和 ETA → UI 更新
```

### 边界情况

- ffmpeg 输出格式异常 → 显示"处理中"但无法精确估算
- 任务取消 → 立即停止进度更新

---

## 模块八：批量队列

### 功能列表

- [x] 多文件排队处理
- [x] 串行执行（避免 GPU/CPU 资源抢占）

### 业务逻辑

```
用户添加多个任务 → 任务入队 → 逐个执行 → 每个任务独立显示进度
```

### 边界情况

- 队列中某个任务失败 → 跳过继续执行下一个，失败任务标记错误
- 用户取消队列 → 停止当前任务，清空剩余队列

---

## 模块九：输出控制

### 功能列表

- [x] 自定义输出路径
- [x] 预估输出文件大小
- [x] 输出文件已存在时的覆盖提示

### 业务逻辑

```
用户设置输出路径 → 检查路径是否已存在同名文件 → 存在则提示覆盖/重命名
预估大小: 根据目标码率和时长计算
```

### 边界情况

- 输出路径无写入权限 → 提示权限错误
- 磁盘空间不足 → 预估大小 > 可用空间时提示

---

## 模块十：错误处理

### 功能列表

- [x] 友好错误提示（面向普通用户）
- [x] 折叠查看原始 ffmpeg 日志（面向技术用户）
- [x] 普通用户看不到 ffmpeg 命令；技术用户可展开查看完整命令

### 业务逻辑

```
ffmpeg 返回非零退出码 → 解析 stderr 提取关键错误 → 显示友好提示
                                      → 用户点击"查看详情" → 展示完整 ffmpeg 日志
```

### 边界情况

- ffmpeg 崩溃无输出 → 显示"处理引擎异常"通用提示
- 未知错误码 → 显示原始错误信息

## 二期功能规划（v0.2.0 — v0.5.0）

- 可视化时间轴预览（拖拽选区裁剪）
- 视频转 GIF（帧率/尺寸可调）
- 字幕硬烧（SRT / ASS）
- 文字水印 / 图片水印
- 多文件顺序拼接
- 截图（单帧 / 按间隔批量）
- 界面语言切换（中文 / 英文）

## 三期功能规划（v0.6.0+）

- 画面裁剪（crop，可视化拖拽）
- 旋转 / 翻转
- 变速（0.25x — 4x）
- 亮度 / 对比度 / 饱和度调整
- 多段截取合并输出
- 左右 / 上下并排合并
- 画中画（PIP）
- 音量调整
