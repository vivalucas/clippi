# 数据库设计

> 本项目无数据库，此文件为占位。Clippi 是纯本地桌面应用，不使用数据库。
>
> 任务队列等状态数据在 Rust 核心库内存中管理，应用关闭后不持久化。
> 如后续引入数据库（如本地 SQLite 持久化任务历史），请按下方模板填写。

## 设计决策

- 不使用数据库的原因：Clippi 定位为轻量级本地工具，无需持久化存储。任务状态在内存中管理即可满足需求。
- 如果未来需要记录处理历史或用户偏好，优先考虑 SQLite（嵌入式，零配置）。
